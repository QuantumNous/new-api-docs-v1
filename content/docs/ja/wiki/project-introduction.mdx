## 概要

New API は、[One API](https://github.com/songquanpeng/one-api) をベースに構築された、次世代の大規模モデルゲートウェイおよび AI 資産管理システムです。このプロジェクトは、さまざまな AI モデルサービスを管理・利用するための統一インターフェースを提供します。

<Callout type="info">
  New API は、OpenAI、Anthropic、Google Gemini、Midjourney など、主要な AI
  プラットフォームとシームレスに統合されています。
</Callout>

### 主な機能

- **統一インターフェース** - 複数の AI プロバイダーに対応する単一の API エンドポイント
- **セキュアな管理** - 高度な認証と認可
- **使用状況分析** - 包括的な追跡とレポート
- **高性能** - 速度と信頼性のために最適化
- **マルチテナント対応** - チームや企業に最適

## ライセンス

New API は、**使用ベースのデュアルライセンス（Usage-Based Dual Licensing）**モデルを採用しています。

<Callout type="warn">
  New API を使用する前に、[プロジェクト
  LICENSE](https://github.com/QuantumNous/new-api/blob/main/LICENSE)
  で完全なライセンス条項をお読みください。
</Callout>

### オープンソースライセンス（AGPLv3）

AGPLv3 の下では、個人およびオープンソースプロジェクトで New API を自由に使用、変更、配布できます。

**主な条項：**

- 自由に使用、変更、配布可能
- 完全なライセンステキスト：[GNU AGPLv3](https://www.gnu.org/licenses/agpl-3.0.html)
- **中核的義務**：変更を加えてネットワークサービス（SaaS）としてデプロイする場合、AGPLv3 の下で完全なソースコードを提供する必要があります
- **ブランド要件**：すべてのオリジナルブランド、ロゴ、著作権表示を保持する必要があります

<Callout type="error">
  AGPLv3 ライセンスのみを使用する場合、New API
  のブランドを削除または変更することはできません。ホワイトラベル化には商用ライセンスが必要です。
</Callout>

### 商用ライセンス

以下のシナリオでは、商用ライセンスを取得する**必要があります**：

**シナリオ 1：ホワイトラベル化**

- 製品から New API のロゴ、ブランド、または著作権表示を削除または変更したい場合。

**シナリオ 2：クローズドソース SaaS**

- New API を変更し、変更内容をオープンソース化せずにサービスとして提供したい場合。

**シナリオ 3：企業要件**

- 組織のポリシーで AGPLv3 ソフトウェアが禁止されている場合、または OEM 統合権が必要な場合。

**シナリオ 4：商用サポート**

- 公式技術サポート、SLA、商業保証が必要な場合。

**商用ライセンスの取得：**

商用ライセンスに関するお問い合わせは、[support@quantumnous.com](mailto:support@quantumnous.com) までメールでご連絡ください。

### 貢献

New API へのすべての貢献は AGPLv3 の下でライセンスされ、オープンソースおよび商用配布版の両方に含まれる可能性があります。

**貢献することにより：**

- あなたのコードは AGPLv3 の下でこのプロジェクトとすべてのユーザーにライセンスされます
- あなたの貢献は商用ライセンス配布版に含まれる可能性があります
- あなたは貢献の著作権を保持します

### 追加条項

- 商用ライセンスの条項、条件、価格は正式な契約によって管理されます
- ライセンスポリシーは公式チャンネル（GitHub、ウェブサイト）を通じて更新される場合があります
- 紛争については、正式な [LICENSE](https://github.com/QuantumNous/new-api/blob/main/LICENSE) ドキュメントを参照してください
